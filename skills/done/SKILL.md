# done スキル

指定されたタスクの `status` を `done` に更新し、関連する Spec があれば受入条件の達成状況を確認する。

## 引数の処理

`$ARGUMENTS` を解析:

- `p:<project>` → プロジェクト名（必須）
- 残りの文字列 → タスクの slug または部分一致キーワード（必須）

例:
- `/done p:homepage-renewal hero-image-design` → slug 完全一致で検索
- `/done p:homepage-renewal ヒーロー画像` → タスクタイトルの部分一致で検索

引数バリデーション:

- `p:` が省略された場合 → 「プロジェクト名を指定してください。例: `/done p:my-project task-slug`」と通知して終了
- タスク指定が空の場合 → 「完了にするタスクを指定してください」と通知して終了

## 処理フロー

### ステップ1: タスクの検索

1. Glob で `/Users/apple/YourOS/Projects/<project>/Tasks/*.md` を検索（`_template.md`, `_index.md` は除外）
2. 以下の順序でマッチを試みる:
   - ファイル名に slug が完全一致するもの
   - ファイル名に slug が部分一致するもの
   - タスクタイトル（`# タスク:` の後ろ）にキーワードが部分一致するもの
3. マッチ結果:
   - 0件 → 「該当するタスクが見つかりません: (キーワード)」と通知して終了
   - 1件 → そのタスクを処理
   - 複数件 → 候補一覧を表示してユーザーに選択を求める

### ステップ2: タスクの状態更新

1. Read でタスクファイルを読み込む
2. 現在の `status` を確認:
   - `status: done` の場合 → 「このタスクは既に完了しています」と通知して終了
   - `status: archived` の場合 → 「アーカイブ済みのタスクです。変更できません」と通知して終了
3. YAML frontmatter を以下のように更新:
   - `status: active` → `status: done`
   - `updated: YYYY-MM-DD` → 今日の日付に更新
4. Write でファイルを上書き

### ステップ3: Spec 受入条件の照合（任意）

タスクの frontmatter に `spec:` が設定されている場合:

1. Spec ファイルを Read で読み込む
2. `## 受入条件` セクションのチェックリストを表示
3. 各条件について達成状況をユーザーに確認:

```
関連 Spec の受入条件を確認します:

- [ ] ファーストビューに新しいヒーロー画像が表示される → 達成していますか？
- [ ] モバイルでもレイアウトが崩れない → 達成していますか？
```

4. 全条件が達成されている場合 → 「全ての受入条件を満たしています」と通知
5. 未達成がある場合 → 「N件の受入条件が未達成です。タスクは完了にしましたが、確認してください」と警告

### ステップ4: 完了通知

以下を通知:

```
タスクを完了にしました:
- タスク: タスクタイトル
- プロジェクト: project-name
- ファイル: パス
- Spec照合: N/N 条件達成（Spec がある場合）
```

**重要**: すべての出力は日本語で行うこと。
